Lorsque vous utilisez Crowd pour une authentification unique (SSO), vous pouvez spécifier le nom du cookie SSO pour chaque application. Dans la configuration standard, Crowd utilisera un seul nom de cookie par défaut pour toutes les applications connectées à Crowd. Vous pouvez surcharger la valeur par défaut avec votre propre nom de cookie. En plus de vous permettre de définir le nom du cookie SSO, cette fonctionnalité vous permet également de diviser vos applications en différents groupes SSO. Par exemple, vous pouvez utiliser un jeton SSO pour vos sites web publics et un autre pour vos sites web internes.
