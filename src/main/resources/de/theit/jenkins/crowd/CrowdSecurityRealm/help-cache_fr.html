Certains serveurs Crowd peuvent être lents ou limiter les requêtes du client. Dans de tels cas, l'activation de la mise en cache peut améliorer les performances de Jenkins avec le risque d'une propagation retardée des changements opérés par les utilisateurs depuis Crowd et d'augmenter l'utilisation de la mémoire sur le nœud du contrôleur.
Remarque : La configuration par défaut est de laisser le cache désactivé.
